# 패턴
1. 생성 패턴 
  > 객체의 생성과 변경이 전체시스템에 미치는 영향을 玄統 하고, 코드의 유연성을 높 힘.


2. 구조 패턴
  > 프로그램 내의 Data Structure 나 interface 구조 등, 프로그램구로 설계시 활용될 수 있는 패턴
3. 행위패턴
  > 반복적으로 사용되는 객체들의 상호작용을 패턴화시킨 것.
